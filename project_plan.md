这是一个学习向的微前端项目，主要用于学习微前端的原理和实现。以及处理常见微前端问题。

Demo 项目设计（学习向）

- 总体形态 ：一个 Host（基座应用）+ 3～4 个子应用（Micro Apps）+ 公共组件/工具库
- 技术选择 ：Host 用 React 或 Vue，子应用混合（例如 React + Vue + Vanilla），突出“多技术栈”
- 场景主题 ：电商管理后台（常见、可扩展）
  - 子应用 A：商品管理（列表、编辑、详情）
  - 子应用 B：订单管理（列表、状态流转）
  - 子应用 C：用户/权限管理
  - 子应用 D：可选“可视化/报表”或“营销活动”
- 路由策略 ：主路由 + 子应用内部路由，支持子应用独立运行与集成运行
- 通信机制 ：全局事件总线 + Host 统一状态（或共享状态库）
- 构建与部署 ：子应用独立构建、独立发布，Host 运行时加载
必须覆盖的微前端能力点

- 子应用加载方式 ：运行时加载（动态 import/HTML entry），并保留“静态集成”对比
- 生命周期管理 ：加载、挂载、卸载、错误处理
- 路由隔离 ：主/子路由的互不干扰，支持子应用内跳转
- 样式隔离 ：CSS Scope / Shadow DOM / BEM 约定 + 冲突测试
- JS 沙箱 ：全局变量污染隔离、事件处理隔离
- 依赖共享 ：React/Vue 等共享依赖策略（版本冲突处理）
- 公共能力抽离 ：登录态、鉴权、菜单、埋点、国际化、主题
- 性能策略 ：预加载/懒加载、资源缓存、首屏优化
- 独立运行 ：子应用可脱离 Host 单独启动开发
- 降级方案 ：子应用不可用时的错误提示与兜底
推荐的 Demo 结构

- Host
  - 全局导航、登录态、统一布局、菜单权限、子应用加载
- Micro Apps
  - A：商品 CRUD（组件复杂度适中）
  - B：订单流转（状态机 + 表单）
  - C：权限与用户（典型 RBAC）
  - D：图表（体现第三方依赖）
常见微前端问题清单

- 样式冲突 ：全局 CSS 互相覆盖，第三方组件样式污染
- 全局变量污染 ：window 上的命名冲突、事件注册遗留
- 路由冲突 ：hash/history 混用、子应用刷新 404
- 版本冲突 ：React/Vue 不同版本冲突、重复依赖体积膨胀
- 状态割裂 ：子应用间数据共享困难、状态一致性问题
- 性能问题 ：首次加载资源巨大、重复下载依赖
- 构建问题 ：不同构建工具输出不一致、资源路径错乱
- 部署与发布 ：子应用独立发布后 Host 未更新入口
- 鉴权问题 ：权限边界不清晰、子应用绕过 Host
- 通信问题 ：事件爆炸、消息顺序与幂等难管理
- 错误隔离 ：某个子应用崩溃影响整体页面
- 调试体验差 ：本地联调繁琐、日志难追踪
